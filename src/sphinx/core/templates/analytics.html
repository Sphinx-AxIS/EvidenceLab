Analytics

Use Analytics as your main hunting workspace. Start broad, narrow to suspicious patterns, then pivot to a specific record for detailed inspection before creating a rule.

ATT&CK Starter Filters

  • The Windows ATT&CK Starter Filters panel is a guided selector that narrows from Tactic to Technique to a specific EventID starter.
  • Each final EventID choice applies a record type plus starter EventID and channel filters to the existing analytics query tools.
  • The selector is populated from the local Windows ATT&CK mapping dataset bundled with the platform.
  • These presets are starting points, not finished detections. Inspect the resulting records carefully before deciding to create a Sigma rule.

Suggested Workflow

  • Start with an ATT&CK preset or click a record type card to open that evidence in Analytics.
  • For Windows event logs, Analytics exposes top-level fields like Channel and EventID plus nested fields such as EventData.TargetUserName and System.TimeCreated.SystemTime.
  • You can filter the native ts timestamp column directly, including partial timestamp text such as 2025-09-02 11:2 with the contains operator.
  • Large field values such as Windows EventData JSON are rendered as literal escaped text in the results grid so clickable filter cells cannot leak HTML into the displayed evidence.
  • If an Analytics results table is wider than the page, a fixed bottom scrollbar stays available so you can move across columns without jumping to the bottom of the results area.
  • Use Browse, Value Counts, Relationships, or Time Series to understand what is rare, repeated, or suspicious.
  • If you switch tabs or launch a new query before the previous one finishes, EvidenceLab now ignores stale results so an older Correlate search cannot overwrite a newer Browse query.
  • Open a specific record from the results table to inspect it in context.
  • Only after the event is understandable should you pivot into Sigma or Suricata authoring.
