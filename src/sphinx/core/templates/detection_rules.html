Detection Rules

Detection rules are generated from investigation findings. They encode behavioral patterns into machine-readable signatures that can automatically detect similar attacks in future evidence.

Rule Types

  • Suricata — Network detection rules. Run against PCAP files during ingest. Match patterns in network packet payloads (commands, protocol anomalies, traffic behaviors).
  • Sigma — Log detection rules (YAML format). Compiled to SQL and run against Windows event log records during ingest. Can also be exported for use in external SIEMs.

Rule Lifecycle

  • Pending Review — Newly generated by the LLM. Click "Review" to inspect and edit.
  • Approved — Reviewed and accepted. Ready to deploy.
  • Deployed — Actively running against new evidence during ingest. Can be re-deployed after edits.
  • Rejected — Reviewed and discarded (inaccurate, too broad, etc.).

Actions

  • Review — View and edit the rule content. You can modify the rule before approving.
  • Deploy — Makes the rule active. Suricata rules are written to the rules directory; Sigma rules are compiled to SQL.
  • Re-deploy — Updates the deployed version after you've edited the rule.
  • Export — Download the rule as a file (.rules for Suricata, .yml for Sigma) for use in external tools.
