Findings

Findings are the results of LLM-powered investigations. Each finding describes what the investigation discovered, with links back to the specific evidence records that support it.

Generating Detection Rules

  • Check the box next to one or more findings you want to create detection rules from.
  • Click Generate Detection Rules from Selected at the top.
  • The system will analyze the evidence behind each finding and use the LLM to generate behavioral Suricata rules (for network evidence) or Sigma rules (for Windows event logs).
  • Generated rules appear on the Detection Rules page for review, editing, and deployment.

Deleting Findings

  • Click the trashcan icon to the left of a finding to delete it.
  • You can only delete findings generated by tasks you created.
  • Admins can delete any finding.
  • Deletions are permanent and cannot be undone.

What makes a good finding for rule generation?

  • Findings with Evidence IDs listed — the LLM needs the actual evidence records to write accurate rules.
  • Findings that describe behavioral patterns (e.g., "attacker used base64-encoded payloads via SSH") rather than just IOCs (e.g., "attacker IP was 1.2.3.4").
  • Findings with MITRE ATT&CK tags — these help the LLM write properly categorized rules.
