User Management

Create and manage user accounts. Each user has a role that determines what they can access.

Roles

  • Admin — Full platform access. User management, system configuration, data management.
  • Case Manager — Create and manage cases, assign analysts, view all cases.
  • Analyst — Investigate assigned cases, run tasks, create findings. Must be assigned to specific cases.
  • LLM Agent — Service account for automated AI investigation tasks. Connects to the database with a restricted role (SELECT-only on evidence tables) and receives a short-lived JWT scoped to the specific case(s) being investigated.

LLM Agent Service Account

An llm_agent account is automatically created at startup. When an investigation task runs, the system mints a 30-minute JWT scoped to the task's case (Investigator mode) or source cases (Correlator mode). The REPL container connects to the database as sphinx_repl with row-level security enforcing case boundaries.

Actions

  • Manage — View user details, change role, reset password, assign/unassign cases.
  • Deactivate — Disable a user account without deleting it. They can no longer log in.
