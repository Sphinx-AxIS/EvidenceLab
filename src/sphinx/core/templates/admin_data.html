Data Management

This page provides administrative tools for deleting evidence, cases, and ingest job data. All deletions are permanent and cannot be undone.

Deletion Options

  • Delete Records by Ingest Job — Remove all evidence records that were created by a specific PCAP or file ingest job. Useful when an ingest produced bad data and needs to be re-run. The job record itself is kept (marked as "deleted") for audit purposes.
  • Delete Case Evidence — Remove all evidence from a case (records, entities, findings, tasks, worklog, jobs) but keep the case itself. Use this to reset a case for re-investigation.
  • Delete Entire Case — Permanently remove a case and everything tied to it: evidence, tasks, findings, notes, user assignments, and the case record itself.

Important Notes

  • Detection rules generated from a case's findings are not deleted — they are independent global assets.
  • Only jobs with linked records show the "Delete Records" button. Jobs without the link (ingested before job tracking was added) can only be cleaned up via the case-level options.
