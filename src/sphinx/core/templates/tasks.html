Investigation Tasks

Tasks are investigation questions that the LLM answers by writing and executing Python code against the evidence database.

How Tasks Work

  • Click New Task to create an investigation question (e.g., "What happened on this host and what techniques were used?").
  • The system runs pre-computation (analytics shortcuts), then starts the LLM investigation loop.
  • The LLM writes Python code, executes it in a sandbox, reads the results, and iterates — up to the max steps you set.
  • Each step is logged in the worklog (click a task to see the full trace).
  • When done, the LLM produces a finding with a summary and evidence citations.

Task Status

  • Pending — Created but not yet started.
  • Running — LLM investigation is in progress.
  • Done — Investigation complete. Check Findings for results.
  • Failed — Something went wrong. Click the task to see the worklog for error details.
