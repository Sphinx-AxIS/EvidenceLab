Dashboard

The dashboard is your central hub for managing investigations. Select a case to see its evidence summary, active tasks, and background jobs.

Investigation Modes

  • Investigator — Focus on a single case. The LLM only sees evidence from the selected case. This is the default mode for most investigations.
  • Correlator — Cross-case analysis. Select multiple source cases to analyze together. The LLM can query evidence across all selected cases to find shared indicators, common attack patterns, and campaign-level connections. Findings are saved to a dedicated correlation case.

Case Settings

  • HOME_NET — Define the internal network ranges for this case (CIDR notation, comma-separated). Suricata uses this to determine traffic direction when running rules against PCAPs. If not set, the default from the system configuration is used.
  • Victim IPs — Specific IP addresses of compromised or targeted hosts. These are included in the LLM's system prompt so it knows which hosts to focus on during investigation.
  • Description — Brief description of the incident. Used in reports and LLM context.
  • Changes take effect immediately — no need to restart containers. Set these before ingesting PCAP files so Suricata uses the correct HOME_NET.

Dashboard Sections

  • Records by Type — Evidence ingested into the case, grouped by source (Suricata alerts, Zeek connections, tshark streams, Windows event logs, etc.). Click "View" to browse individual records.
  • Background Jobs — Long-running operations like PCAP ingest. Shows progress, status, and record counts.
  • Loaded Plugins — Active analysis plugins providing ingest, detection, and analytics capabilities.
