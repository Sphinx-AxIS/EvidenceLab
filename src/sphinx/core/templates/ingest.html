Ingest Evidence

Upload evidence files to add them to the current case. The system parses the data, stores it as structured records, and automatically extracts entities (IPs, domains, hashes, etc.).

Supported Formats

  • PCAP / PCAPNG — Network capture files. Automatically processed by Suricata (IDS alerts), Zeek (connection metadata, protocol logs), and tshark (TCP stream reconstruction). Select "PCAP Upload" and choose your capture file.
  • EVTX — Native Windows Event Log files. Select "Windows Event Log (EVTX)" and upload a `.evtx` file; the system will auto-detect channel, preserve the original event timestamp as the record timestamp, and route events to the existing Windows log record types.
  • JSON / JSONL — Structured log files. Select the appropriate record type (e.g., win_evt_security for Windows Security logs) and upload a JSON array or newline-delimited JSON file.

What Happens During Ingest

  • PCAP ingest runs in the background — you can monitor progress on the Dashboard.
  • The Ingest page now shows a form-level upload progress bar for every ingest mode. JSON, EVTX, and PCAP uploads can all poll server-side job progress, and EVTX/PCAP jobs show processed counts when totals are available.
  • PCAP processing runs inside the isolated REPL container and writes derived records back into the case through an internal ingest path.
  • JSON ingest is immediate — records are inserted and you're redirected back with a count.
  • Deployed detection rules (Suricata and Sigma) automatically run against newly ingested evidence.

Advanced Troubleshooting

  • The page includes a collapsed Available Handlers section for analysts or operators who want to see the registered ingest handler names.
  • This list is mostly useful for troubleshooting or understanding internal record types. Most analysts can ignore it.
